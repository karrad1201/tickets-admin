# ADR 0001: Tech Stack — Next.js + shadcn/ui

## Status

Accepted

## Context

Нужна веб-админка для платформенного администратора. Требования:

- небольшая команда, быстрый старт;
- SSR для первичного рендера таблиц (SEO не нужен, но скорость первой загрузки важна);
- готовый UI kit, совместимый с Tailwind CSS;
- деплой на Vercel без дополнительной инфраструктуры.

Рассматривались варианты:

| Вариант | Проблема |
|---------|---------|
| React (Vite) + React Router | Нет SSR, нет встроенного routing, нужен отдельный BFF |
| Remix | Меньше экосистема, команда не знакома |
| Next.js Pages Router | Устаревший, App Router — текущий стандарт |
| Next.js App Router | ✓ |

Для UI рассматривались:

| Вариант | Проблема |
|---------|---------|
| MUI | Тяжёлый, стиль не совместим с Tailwind |
| Ant Design | То же самое |
| shadcn/ui | ✓ — компоненты копируются в репо, полный контроль, Tailwind-native |

## Decision

- **Next.js 15** с App Router.
- **TypeScript** строгий режим.
- **Tailwind CSS** для стилизации.
- **shadcn/ui** как источник базовых компонентов (копируются в `src/components/ui/`).
- **TanStack Table** внутри `DataTable` для таблиц с сортировкой и пагинацией.

## Consequences

Плюсы:

- Server Components позволяют делать fetch напрямую на бэкенд без клиентского waterfall.
- shadcn/ui даёт готовые Dialog, Form, Table без lock-in на внешнюю библиотеку.
- Vercel деплоит Next.js без конфигурации.

Минусы:

- App Router — относительно новый, некоторые паттерны ещё устоялись не до конца.
- shadcn/ui компоненты нужно обновлять вручную (нет npm package).
